Supabase update memo (soccer schedule app)

This app now uses a category master for schedule classification.
Please apply the SQL updates in `src/sql/table/table.sql` or run the migration steps below in Supabase SQL Editor.

1) Add classification master table
- Create `public.category_master` for schedule categories:
  - `practice` = 練習
  - `match` = 試合
  - `event` = イベント

2) Migrate `public.schedule`
- Replace the old `title` column with `category_id`.
- Add foreign key from `public.schedule.category_id` to `public.category_master(id)`.
- If existing data is already in production, migrate before dropping the old column:
  ```sql
  create table if not exists public.category_master (
    id bigint generated by default as identity primary key,
    category_code text not null unique check (category_code in ('practice', 'match', 'event')),
    category_name text not null,
    display_order integer not null default 1,
    is_active boolean not null default true,
    created_at timestamptz not null default now(),
    updated_at timestamptz not null default now()
  );

  insert into public.category_master (id, category_code, category_name, display_order, is_active)
  values
    (1, 'practice', '練習', 1, true),
    (2, 'match', '試合', 2, true),
    (3, 'event', 'イベント', 3, true)
  on conflict (id) do update
  set category_code = excluded.category_code,
      category_name = excluded.category_name,
      display_order = excluded.display_order,
      is_active = excluded.is_active,
      updated_at = now();

  alter table public.schedule
    add column if not exists category_id bigint references public.category_master(id) on delete restrict;

  update public.schedule
  set category_id = case
    when lower(coalesce(title, '')) like '%match%' or title = '試合' then 2
    when lower(coalesce(title, '')) like '%event%' or title = 'イベント' then 3
    else 1
  end
  where category_id is null;

  alter table public.schedule
    alter column category_id set not null;

  alter table public.schedule
    drop column if exists title;
  ```

3) Recheck RLS and policies
- Enable RLS for the new table:
  ```sql
  alter table public.category_master enable row level security;
  ```
- Add policies:
  - authenticated users can read category master
  - admin can manage category master

4) Seed / master data
- Make sure these three master rows exist:
  - 練習
  - 試合
  - イベント
- Keep `display_order` as `1, 2, 3`.

5) Frontend query impacts
- Frontend now reads:
  - `public.category_master`
  - `public.schedule.category_id`
- Old `schedule.title` is no longer used by the admin screen.

6) Recommended verification in Supabase
- Confirm `category_master` has 3 rows.
- Confirm `schedule.category_id` is filled for all existing records.
- Confirm admin users can insert/update/delete schedules after the migration.
- Confirm players/admins can still read schedules and attendance with current RLS.
