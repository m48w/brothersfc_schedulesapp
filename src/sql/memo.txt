Supabase update memo (Brothers FC schedules app)

Apply the latest schema in `src/sql/table/table.sql` if this is a new project.
If you are updating an existing Supabase project, make the changes below before using the new admin screen.

1) Update `public.user`
- Add `password text`
- Add `created_at timestamptz default now()`
- Add `updated_at timestamptz default now()`
- Existing auth still uses Supabase Auth. The `password` column is only an app-side reference field.

2) Update `public.location_master`
- Add `location_type text not null check (location_type in ('stadium', 'event'))`
- Set every existing location row to either `stadium` or `event`

Example:
```sql
alter table public.location_master
  add column if not exists location_type text;

update public.location_master
set location_type = 'stadium'
where location_type is null;

alter table public.location_master
  alter column location_type set not null;
```

3) Update `public.schedule`
- Add `vote_deadline timestamptz not null`
- Make `location_id` required (`not null`)
- Existing player voting now closes at `vote_deadline`, not at `schedule.start_time`

Example migration:
```sql
alter table public.schedule
  add column if not exists vote_deadline timestamptz;

update public.schedule
set vote_deadline = (schedule_date::timestamp + coalesce(start_time, time '00:00'))
where vote_deadline is null;

alter table public.schedule
  alter column vote_deadline set not null;

update public.location_master
set location_type = coalesce(location_type, 'stadium');

alter table public.schedule
  alter column location_id set not null;
```

4) Seed and master data
- Re-run `src/sql/sample/sample.sql`
- Recheck `src/sql/sample/master.json`
- Make sure schedule sample rows include:
  - `vote_deadline`
  - `location_id`
  - event schedules pointing to `event` locations when needed

5) Frontend impacts
- Admin schedule form now requires:
  - vote deadline
  - location type
  - location
- Player attendance button availability now depends on `vote_deadline`
- Admin header now uses `react-icons` and mobile navigation matches the player screen style

6) Recommended verification in Supabase
- `user.updated_at` exists
- `location_master.location_type` is filled for all rows
- `schedule.vote_deadline` is filled for all rows
- `schedule.location_id` has no null rows
- Admin can create and edit schedules with the new required fields
- Players can update attendance before the vote deadline and cannot update it after the deadline
